# rl_env_constraints.txt — PINNED transitive versions for the gpu-rl RL stage.
#
# WHY THIS FILE EXISTS (the gpu-rl-00220aac regression, 2026-06-29):
#   The vLLM-fork + flash-attn WHEELS are ABI-pinned (prebuilt-wheelhouse, locked
#   to torch 2.11.0 / cu128 / cp312), but the rl-stage `uv pip install`s of the RL
#   env were NOT version-locked. A later rebuild therefore silently re-resolved
#   DIFFERENT transitive versions of NCCL-affecting deps (most notably the
#   torch-bundled `nvidia-nccl-cu12`, plus `ray` / the `nvidia-*-cu12` CUDA libs),
#   which REGRESSED the 32-rank `default_pg` NCCL rendezvous: the rebuild
#   gpu-rl-00220aac (@sha256:65b07cec) died TWICE at build_models /
#   init_weight_sync_state with
#     DistBackendError: store->get('0') wait timeout after 1200000ms.
#   The last-known-good image gpu-rl-81045a29
#   (@sha256:648199885a1e35cebef029ab2993d527d88d3a2c7f3b30c1313990c81452c99e)
#   clears those phases cleanly (verified live: rl-131k-30b-stepfix3, 0 rendezvous
#   failures).
#
# THE RULE (also in .claude/skills/build-gpu-rl-image-iris/SKILL.md):
#   Every gpu-rl rebuild MUST install the rl stage with
#     uv pip install --constraint docker/rl_env_constraints.txt ...
#   so the resolver reproduces the EXACT known-good versions. Regenerate this file
#   from the CURRENT good image's `uv pip freeze` ONLY when an intentional dep bump
#   is wanted — never let the transitive set float.
#
# SOURCE OF TRUTH: `uv pip freeze` of /opt/openthoughts/envs/rl in the known-good
#   image gpu-rl-81045a29 (@sha256:648199...c99e), extracted via iris job
#   gpurl-freeze-extract2 on 2026-06-29 (Python 3.12.13).
#
# EXCLUDED (handled separately by Dockerfile.gpu-rl, NOT constrainable as a
#   name==version pin — they are git/file/editable installs):
#     - flash-attn, vllm        -> the prebuilt wheels (COPY --from=wheels; --no-deps)
#     - torchtitan              -> `git+...@a1fdd7e`  (--no-deps, step 4a)
#     - harbor                  -> `git+...@<HARBOR_COMMIT>` (--no-deps, step 4b)
#     - dynamic-semaphore       -> git+ (transitive of harbor; rides its install)
#     - skyrl-gym, skyrl-train  -> `-e /opt/skyrl/...` editable (step 4)
#     - openthoughts            -> `-e /opt/openthoughts` editable (step 4)
#   (Their PyPI deps ARE pinned by the entries below; only the editable/URL
#    distributions themselves are omitted, since a constraints file forbids URLs.)
#
# ALSO EXCLUDED — the boto/s3 client cluster (aiobotocore / botocore / boto3 /
#   s3transfer). The freeze captured an INTERNALLY-INCONSISTENT pair that a single
#   uv solve rejects: aiobotocore==3.7.0 requires botocore<1.43.1, yet the freeze
#   also has botocore==1.43.36. The good image reached that state SEQUENTIALLY —
#   the Dockerfile's later explicit `uv pip install boto3` (step 4) bumped botocore
#   to 1.43.36 AFTER s3fs->aiobotocore had resolved the older one. Constraining
#   both at once is unsatisfiable, so we leave the boto cluster unpinned: s3fs is
#   pinned (skyrl-train's exact dep) and pulls aiobotocore==3.7.0, then step 4's
#   `boto3` install re-pins botocore==1.43.36 — reproducing the known-good end
#   state. These libs are S3/AWS clients, irrelevant to the NCCL regression.
